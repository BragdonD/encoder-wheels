ESP8266 Web Server - Le contrôleur de moteur DC - NodeMCU

Ce projet a été réalisé par Ibrahim KEBE, Thomas DUCLOS, Hugo VIDAL, Quentin FEVE. Nous sommes tous les 4 étudiants à l'ECE Paris en ING2.

Présentation

Ce quatrième projet d’électronique a été très différent des trois précédents, car il était en relation direct avec notre projet annuel l'ECE CUP.
La durée du projet est d'environ un mois et demi.
Lors de ce projet, nous devions mettre en place de nouvelles fonctionnalités pour notre robot à l’aide des connaissances acquises avec la matière Systèmes Bouclés.
Nous avions pour objectif de développer les fonctionnalités suivantes :

  • Réguler la vitesse du robot.

  • Transformer le robot en gyroscope.

  • Communiquer avec un protocole wifi avec le robot.

  • Mettre en place une page web pour contrôler le robot.

Nous attestons que ce travail est original, qu'il est le fruit d'un travail commun à la squad et qu'il a été rédigé de manière autonome.

Contexte et problématique

Contexte

Le gyropode est une invention de Dean Kamen, Susan D Dastous, Robert Duggan et Michael Guay. Le gyropode a été inventé en l’an 2000 et son premier modèle fût le Segway TPI167. Ce premier modèle a été précurseur du gyropode a 6 roues inventé en avril 2003 pour les handicapés. Vente après-vente, en 2006 la technologie leensteer qui permet de faire tourner le gyropode grâce au poids du corps et aujourd’hui on s’en sert dans la vie de tous les jours pour nos déplacements quotidiens.
Dans notre projet le gyropode sera utile pour stabiliser notre robot afin d’avoir des déplacements plus précis.

Problématique

L’objectif de ce projet est de pouvoir stabiliser le robot et de le contrôler via une interface web. Sur cette interface on doit pouvoir régler la vitesse des moteurs afin de diriger le robot dans ses déplacements. Il faut donc rendre les moteurs plus précis et pour se faire il est nécessaire d’intégrer un PID il faut donc pouvoir le régler aussi à l’aide du site.
Comment régler la précision des moteurs pour rendre le robot parfaitement stable ?

Développement

Ce projet est divisé en quatre modules différents. En effet, pour mener à bien notre projet nous devions réaliser les parties les unes après les autres. Cette démarche nous a forcé à nous organiser, à communiquer et c’est pour cela que la répartition des tâches a été très importante.
Vous trouverez dans les paragraphes suivants la description de quatre différents modules et la manière avec laquelle nous avons procédé pour remplir toutes nos missions.

Module 1 : Le contrôleur de moteur DC

Ce module consiste en l’écriture d’un code permettant de contrôler les moteurs DC du robot. Pour ce faire, nous avons fixé les roues codeuses sur les moteurs et avons établi un code qui calcul la vitesse angulaire à l’aide d’une fonction d’interruption (voir algorigramme ci-dessous).

La fonction d’interruption incrémente le nombre de trous de la roue codeuse détectés par le capteur. Ce nombre de trou est ensuite diviser par 20 qui est le nombre de trou présent sur la roue codeuse. Cette fonction de calcul est placée à l'intérieur d'un timer appelé toutes les secondes pour obtenir une vitesse en tour par seconde. tours par le temps écoulé.

Bilan

Etat d'avancement

Nous sommes aujourd’hui à la fin du projet voici un résumé de notre avancement. Nous sommes parvenus à compléter les trois premières parties du projet c’est à dire la mesure de la vitesse des moteurs ; l’interface Web ; Le PID pour chaque moteur. Malheureusement nous n’avons pas réussi à finir la partie quatre sur le Gyropode.

Effectivement, nous avons essayé de réutiliser le tutoriel fournit pour ce module, mais nous n'avons pas réussi à le faire fonctionner avec l'ESP8266. Nous avons même essayé de le faire fonctionner avec une Arduino Nano mais nous avons obtenu le même résultat. De plus, nous avons essayé de changer de capteurs sans succès.

Nous pensons donc que le problème est un problème hardware pour les capteurs.

Pertinence de la solution technique

Ce projet a ses limites, en effet le matériel que nous utilisons ne nous permet pas de faire d’avantage d’innovations sur notre robot. Par exemple l’esp ne possède pas assez de pin pour rajouter plusieurs fonctionnalités.
Pour aller au-delà des missions qui nous étaient fixées nous avions pensé à améliorer la détection de l’environnement.
